Future Enhancements

  2. Future Enhancements

Reorder restock quantity selection

The quantity that needs to be ordered will differ from time to time so the manager should be able to order the exact quantity that he needs for the establishement instead of a fixed quantity.

The content will be available in deployment 3.0

Current deployment version 2.5

Intergration to a live data feed

Since the data in a supermarket are not kept in files it is necessary to enable a live data feed that can update the database which provides data for this system.

The content will be available in deployment 4.0

Current deployment version 2.5

Sales Prediction

Lack of timed data prevented this feature from being implemented. However with proper data it is possible to predict which items will be sold and their respective time periods. tracking of slow moving and fast moving items ca be enahnced greatly with this enabled.

The content will be available in deployment 4.0

Current deployment version 2.5

Item placement Optimization

As discovered from the research optimizing the placement of items in the retail establishemnts changes and increases the exposure of the establishment to the customers therefor yielding better satisfied customers and an optimized shopping experience

The content will be available in deployment 4.0

Current deployment version 2.5

Database Distribution

As in the research the retail establishemnts are not standalone outlets they are interconected through network to regional office's and then to a central head office. The monitoring process of the stocks from a HQ must be enabled in order to achieve full realization of the retail goals.

The content will be available in deployment 4.0

Current deployment version 2.5

Categorical Optimization

Aquiring proper categories of items and their subcategories and relavant information will provide the base for a better optimization in category mining and layout optimization.

The content will be available in deployment 4.0

Current deployment version 2.5
